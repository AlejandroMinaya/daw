Cuestionario General

¿Cuál es la diferencia entre Internet y la World Wide Web?

La Internetsimplemente se refiere a la red mundial entre computadoras. Dígase las conexiones entre diferentes servidores, clientes, ISPs, etc. Por otro lado, la World Wide Web describe el subconjunto de interconexiones entre servidores Web. Originalmente la idea de los servidores Web, era simplemente poner información a disposición de las computadoras en la Internet. [2]

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

  1. GET

    Típicamente se utiliza para recuperar información. [1]

  2. HEAD

    Sirve para solo obtener la cabecera de la información recuperada. [1]

  3. POST

    Nos permite enviar información para generar algún tipo de cambio o acción del lado del servidor. [1]

  4. PUT

    Nos permite colocar un recurso en el servidor sin riesgo a la duplicación. [1]

  5. PATCH

    Sirve para realizar la modificación parcial de algún recurso. [1]

  6. DELETE

    Se utiliza para eliminar un recurso del servidor. [1]

¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

Se debe utilizar el método POST para no exponer esos datos sensibles al público en general. De utilizar GET, por ejemplo, se muestra esta información en el URL.

¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

Se utiliza el método GET.

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

Significa que el servidor pudo responder a nuestro request . Representa un "OK" y no significa que haya ocurrido un error.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

Sí, esto significa que hay un recurso que no se encuentra en el servidor y se traduce a una mala ruta o que no se subió correctamente el mismo.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

No necesariamente. Esto representa un error interno con el servidor y no tiene que haber sido culpa del desarrollador.

¿Qué significa que un atributo HTML4 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

Representa un elemento al que no se le da seguimiento (support). Por ejemplo, algunos elementos despreciados son: <applet>, <center>, <font>, <dir>, <striket>. [3]

¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

  • El desarrollo de un algorimto para manejar los documentos malformados (que no siguen el estándar). [4]
  • La introducción de elementos semánticos como: <section> y <article>. [4]
  • Mejor soporte para web apps y almacenamiento local. [4]

¿Qué componentes de estructura y estilo tiene una tabla?

  • table - para encerrar lodo lo que formará parte de la tabla.
  • thead - para encerrar lodo lo que formará parte de la cabecerade la tabla.
  • tbody - para encerrar lodo lo que formará parte del cuerpode la tabla.
  • tfoot - para encerrar lodo lo que formará parte del piede la tabla.
  • th - para encerrar la fila de títulos en la tabla.
  • tr - para encerrar una fila en la tabla.
  • td - para encerrar una celda de una fila en la tabla.

¿Cuáles son los principales controles de una forma HTML5?

Con HTML5 se introdujeron nuevos elementos y atributos de los cuales caben destacar:

  • type="email": Le dice al buscador que en el campo se espera un correo.
  • type="date": Le dice al buscador que en el campo se espera una fecha.
  • required: Le dice al buscador que el campo es obligatorio.
  • autocomplete: Le dice al buscador que el campo puede ser autocompletado.

Cuestionario Ciclo de Vida

¿Cuál es el ciclo de vida de los sistemas de información?

Se separa en 7 etapas:

  1. Planeación - Determinar el problema a resolver y estimar lo que se necesita para hacerlo. [5]
  2. Análisis de Requisitos - Se determinan los requisitos funcionales, no funcionales, de información, entre otros para el sistema. [5]
  3. Diseño del sistema - Se desarrollan diagramas y bosquejos para determinar antes de desarrollar como funcionará el sistema. [5]
  4. Desarollo - Se comienza a programar y desarrollar lo que se diseño en la etapa anterior. [5]
  5. Testing - Se prueba lo desarrollado y nos aseguramos que cumpla con lo que se espera. [5]
  6. Implementación - Se pone en producción lo desarrollado para que resuelva el problema planteado al inicio. [5]
  7. Mantenimiento - Con datos del mundo real se ajusta el sistema para hacerlo más eficiente. [5]

¿Cuál es el ciclo de desarrollo de sistemas de información?

Dependiendo de la ténica de desarollo (en espiral, agile, etc). El ciclo de desarollo varia, pero por lo general se guía del concepto de:

Análisis --> Diseño --> Desarollo --> Testing --> Análisis.
