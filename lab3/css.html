Cuestionario General

Como ingeniero de software ¿cuál es tu recomendación sobre el uso de !important en un CSS?

El uso de !important debería ser el último recurso. Tiene mucho poder y hay que tener responsabilidad al usarlo. Si lo usas indiscriminadamente, es evidencia de una mala práctica.

Si se pone una imagen de fondo en una página HTML, ¿por qué debe escogerse con cuidado?

Las imágenes de fondo representa un reto en ciertas ocasiones. Primero se debe escoger con cuidado porque debe tener la resolución y tamaño suficiente para que ocupe correctamente la ventana. Eso puede traer como resultado una imágen pesada que tarda en cargar, lo cuál es en sí otro problema. Por eso a la hora de escoger una imágen para un fondo hay que pensarlo dos veces.

Como ingeniero de software, ¿cuál es tu recomendación al elegir las unidades de un propiedad de estilo entre %, px y pt?

Para un diseño responsivo utilizaría % para los tamaños de los elementos y para sus respectivos márgenes y relleno. Para los bordes utilizaría px, mientras que para las letras de no poder usar rem o vw, utilizaría pt. Al final todo depende de que tan responsivo se quiere realizar el diseño de cuya página (en el mayor de los casos 100% responsiva).

¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio?

Los espacios y saltos de líneas no son imprescindibles para ejecutar el archivo, sin embargo, ocupan espacio y memoria. Al minimizar el archivo se elimina ese espacio extra y el recurso puede cargar más rápido.
